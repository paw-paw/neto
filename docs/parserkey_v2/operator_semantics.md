# NETO ParserKey v2 — Operator Semantics Required by the Initial Corpus

This document fixes the minimum runtime behavior needed to execute the eight ParserKeys in this package.

## Record locators

### `records.row_scan`
Emits one record anchor per row from `start_row` through `end_row`, using `step`.
`anchor_column` identifies the anchor cell used for provenance.

### `records.row_ranges`
Emits one record anchor per row inside each inclusive `{start_row, end_row}` range.

### `records.tile_grid`
Receives an ordered `tiles[]` list. Each tile has an `origin` and `record_row_offsets`.
For every offset, the runtime emits:

- `tile_origin`: the tile's origin cell;
- `record_anchor`: `tile_origin` shifted by that row offset;
- the tile's `tile_id`.

`emit_order = tiles_then_row_offsets` preserves the declared tile order.

## Cell sources

### `cell.column`
Reads a column at the current record row.

### `cell.absolute`
Reads a fixed A1 address from the current source sheet.

### `cell.relative`
Reads a cell relative to `record_anchor` or `tile_origin`.

### `cell.merged_origin`
For the cell at the current row and declared column:

1. if it belongs to a merged range, read the merged range's top-left cell;
2. otherwise read the cell itself;
3. when `fallback = nearest_non_empty_above`, walk upward in the same column.

### `record.row_index`
Returns the 1-indexed Excel row of the current record anchor.

### `context.get`
Returns a named context value after its pipeline has executed.

### `extractor.output`
Returns a named output generated by an extractor in the same record set.

### `literal.value`
Returns the configured literal unchanged.

## Predicates

`predicate.all`, `predicate.any`, `predicate.not_empty`, `predicate.equals`, and
`predicate.matches_regex` evaluate source/value operators against the current record.

## Extractors

### `extract.regex`
Runs patterns in order. Named capture groups become extractor outputs.
Flags use Python regular-expression names such as `IGNORECASE`.

## Transform pipeline

Transforms run left-to-right. A failed transform follows the field's `on_missing` policy.

Required text transforms:

- `text.to_string`
- `text.trim`
- `text.regex_extract`
- `text.regex_replace`

Required datetime transforms:

- `datetime.excel_serial_to_date`
- `datetime.excel_fraction_to_time`
- `datetime.parse_date`
- `datetime.parse_time`
- `datetime.inject_year`
- `datetime.format_date`
- `datetime.format_time`

`datetime.parse_time` with `formats = ["auto"]` must accept both Excel time fractions and strings
such as `10:00`.

## Formula handling

The runtime does not evaluate arbitrary Excel or Google Sheets formulas.

- `cached_value_only`: read the cached result stored in the XLSX;
- `first_available`: try cached value, raw value, then display value;
- if a required cached value is absent, emit the configured warning/error.

## Fallbacks and defaults

A value pipeline's `fallbacks` are attempted in order when the primary source resolves to null/empty.
`on_missing = use_default` emits the field's `default`.

## Overrides

Overrides execute after ordinary field extraction. The runtime must record the override reason in
provenance when enabled.

## Provenance

For every extracted field, internal results should retain:

- source workbook;
- source sheet;
- source cell or merged origin;
- record set;
- transform chain;
- fallback/default/override usage.

## Regression rule

A ParserKey passes its basic regression test only when:

1. it validates against `neto_parserkey_v2.schema.json`;
2. every required operator exists in `neto_operator_catalog_v1.json`;
3. parsed record count equals the key's exact `validation.record_count`;
4. all manifest smoke checks match.
